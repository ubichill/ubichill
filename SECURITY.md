# セキュリティポリシー

## 脆弱性の報告

**公開 Issue には投稿しないでください。** GitHub の **Security → Report a vulnerability**
（Private Vulnerability Reporting）から非公開で報告してください。

報告には以下を含めてください:

- 再現手順（可能なら最小の PoC）
- 影響範囲・想定される被害
- 該当バージョン / コミットハッシュ（画面右下のバージョンバッジ、または `/api/version`）

初回応答の目標は **3 営業日以内**です。修正は原則として最新の `main` に対して提供します。

## 対象範囲

- **コア**（`packages/*`, `charts/*`, Host アプリ本体）: 本ポリシーの対象。
- **サードパーティ製mod**: 各配布元の責任範囲（`COPYING` のmod無保証条項を参照）。
  ただし、modを隔離する**サンドボックス機構そのものの回避（sandbox escape）**は
  コアの脆弱性として扱います。

## 脅威モデル（要点）

Ubichill は**未検証のmodを Web Worker サンドボックスでゼロトラスト実行**します。
特に以下を重大とみなします:

- サンドボックス / capability 境界の回避（許可されていない DOM・ネットワーク・ストレージ・
  他modへの到達）
- 認証・セッションの侵害、他ユーザーへの権限昇格、なりすまし
- サーバ側のインジェクション / SSRF / 認可欠落 / 秘密情報の漏洩

## 責任ある開示

報告いただいた脆弱性は、修正がリリースされるまで非公開で扱います。ご希望に応じて
リリースノートでクレジットします。
